# Functional and Non functional requirements/ requirements gathering

Requirements gathering happens after the business value and the feasibility analysis

# Systems Analysis consists of three basic steps

Review As-is system -> Identify Improvements -> Develop Requirements for the To-Be System

Six activities:

1) Gather detailed information
2) Define functional requirements
3) Define Nonfunctional requirements
4) Develop User Interface Dialogs
5) Prioritize requirements
6) Evaluate Requirements With User

System requirements are all of the capabilities that the new system must meet

Functional requirements - Describes an activity or process that the system must perform (business stuff that people that use the product care about)/ what the system does

Nonfunctional requirements - Describes operational characteristics related to the system (nerdy stuff)/ how well the system does it (performance/ maintainability/ security etc)
