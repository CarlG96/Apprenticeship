# UML Diagrams

4 components: Systems/ Actors/ Use cases/ Relationships

Use case diagrams describe the high-level functions and scope of the system. These diagrams also identify the interactions between the system and its actors.

A system is represented by a square which you can add oval shapes to

External entities are referred to as actors. Actors represent roles which may include human users, external hardware or other systems

Primary actors initiate the use of the system, secondary users will react to the system or action within the system

Three types of relationship: Association - Include, Association - Exclude, Generalisation

Include = mandatory, extend = optional based on a condition
