# Entity Relationship Diagrams

See this url (https://www.youtube.com/watch?v=QpdhBUYk7Kk&t=5s) about cardinalities and relationships between the ERD diagrams (one to one), (one to many), (zero or many) etc

## Primary Keys

Needs to be unique for the table
Needs to be never changing
Can't be null

## Foreign Key

Primary key from another table

## Composite Keys

Taking two attributes that aren't primary keys themselves but when combined can be a primary key

## Data types

Use a three columned table to have PK/FK, the variable name and the data type
